# 영상 포맷 템플릿 (FORMATS.md)

> 파이프라인이 생성하는 Shorts의 **포맷 템플릿** 규칙.
> 이 문서는 [CONTENT.md](./CONTENT.md)(채널·톤·금지 표현 등 공통 가이드)와
> [SPEC.md](./SPEC.md)(기술 스펙)를 **전제로** 하며, 여기에 `포맷 A / C` 레이어를 추가합니다.

---

## 개요

파이프라인은 아래 **2개 포맷 템플릿**을 지원합니다. 기획 단계(큐레이터)에서
주제 성격에 따라 자동으로 하나를 선택하고, 해당 템플릿의 제약을 스크립트 생성에 반영합니다.

두 포맷은 짝으로 설계되어 있습니다.
- **포맷 A**: "무엇이 나왔는지" 알림
- **포맷 C**: "그것을 어떻게 쓰는지" 시연

같은 주제에 대해 **A → C 순서로 시리즈 제작**이 가능합니다.

> 채널(AI / Skin)과 포맷(A / C)은 **독립 차원**입니다. 채널은 주제 도메인,
> 포맷은 전달 방식. 채널별 고정 매핑은 없고 큐레이터가 주제마다 결정합니다.

---

## 포맷 공통 사양 (템플릿 레이어 한정)

> 해상도·프레임레이트·출력 포맷 등 기술 스펙은 [SPEC.md](./SPEC.md) 참고.
> 여기서는 포맷 템플릿 관점에서 **추가로 강제되는 제약**만 명시.

- **자막 정책**:
  - **화면에 박힌 자막 (burned-in): 한국어**
  - **YouTube 캡션 트랙 (CC): 영어 SRT 업로드** → 해외 시청자가 선택, YouTube가 자동번역 베이스로도 사용
  - 이유: Shorts 알고리즘은 업로더 지역(한국) 초기 시청을 기반으로 확산 → 한국인 시청 유지가 글로벌 도달의 전제
- **나레이션-자막 싱크 허용 오차**: 0.2초
- **키워드 강조**: 전체 문장 자막 + 핵심 키워드는 색상 반전 또는 배경 박스로 강조 **필수**
- **BGM**: 미니멀 톤, 나레이션 대비 **-18dB 이하**

## 포맷 A: 정보 전달형 (Informational)

신제품 출시, 기능 업데이트, 툴/프레임워크 소개, 기술 개념 설명.

### 레퍼런스
- https://www.youtube.com/shorts/CnkgNiYa2cM "노션 3.0, AI 에이전트 등장" / 디자인 나침반 / 26초

### 구조 (초 단위, 총 ~26초)

| 구간 | 시간 | 역할 | 대본 글자수 |
|------|------|------|-----------|
| Hook   | 0~3초   | 제품명 + "무엇이 새로운가" | 15자 내외 |
| Body   | 3~18초  | 기능/변화 2~3개 나열 | 80~120자 |
| Value  | 18~23초 | 사용자 이득 | 30~50자 |
| Outro  | 23~26초 | 한 줄 요약 + 가이드형 CTA | 15자 내외 |

### 톤앤매너
- **문체**: "-됩니다", "-입니다" 존댓말 정보 전달형
- **문장 길이**: 짧고 단정적, 평균 20자 이하
- **속도**: 분당 350~400음절
- **금지**: 감탄사, 리액션, "여러분~"
- **고유명사**: 원어 그대로 (Notion, MCP, Claude 등)
- **Outro CTA (가이드형)**: "자세한 활용법은 다음 영상에서 알려드릴게요" 류 — 구독 유도보다 다음 콘텐츠로 연결
  > 구독 강요형("구독 눌러주세요") 대신 가이드형으로 자연스럽게.

### 비주얼
- **배경**: 단색 또는 미니멀 그라데이션
- **전환**: 컷 편집 위주 (페이드/디졸브 지양)
- **필수 소재**: 실제 제품 UI 스크린샷 또는 스크린 레코딩
- **자막 위치**: 중앙 또는 상단 1/3
- **키워드 강조**: 색상 반전 or 배경 박스

### 제목 패턴
- `[제품명/버전], [핵심 변화]` — 예: "노션 3.0, AI 에이전트 등장"
- 길이: 20자 내외

## 포맷 C: 하우투 / 노하우형 (How-to)

실용 팁, AI 툴 활용법, 워크플로우 공유, 프롬프트 팁, 코드 리팩토링 가이드 등.

### 레퍼런스
- https://www.youtube.com/shorts/5Yy_z0CpYZo "자는 동안 제미나이 일 시켜서 100% 자동화하기" / 신영선의 AI탐구 / 22초

### 구조 (초 단위, 총 ~22초)

| 구간 | 시간 | 역할 | 대본 글자수 |
|------|------|------|-----------|
| Hook    | 0~3초   | "~하는 법" / "~하면 ~됩니다" 이익 약속 | 15~20자 |
| Setup   | 3~8초   | 어떤 상황/문제인지 짧게 | 30~40자 |
| How     | 8~18초  | 구체적 방법 (단계/화면 시연) | 60~90자 |
| Result  | 18~22초 | 결과물 + "이제 이렇게 써보세요" | 20~30자 |

### 톤앤매너
- **문체**: "-해보세요", "-하면 됩니다" 실용 가이드형
- **문장**: 짧고 명령형 허용
- **속도**: 분당 380~420음절
- **필수**: 구체적 수치/시간/결과 ("5분만에", "3번 클릭", "한 줄로")
- **금지**: 모호한 표현 ("뭔가 좋아집니다" X)
- **과장 표현 ("100%", "완전 자동화")**: 훅에서만 허용, 본문에 **한계/조건 짧게 언급** (장기 신뢰도)

### 비주얼 (가장 중요)
- **실제 화면 녹화가 핵심**. 클릭 위치/타이핑 내용이 명확히 보여야 함
- **확대 or 하이라이트 박스**로 클릭 지점 강조
- **단계별 짧은 컷** (구간당 2~4초)
- **배경**: 최소화 (화면 녹화가 주인공)
- **자막 위치**: 화면 하단 1/3 고정 (상단은 녹화 화면이 차지)

> 파이프라인은 Puppeteer 기반 자동 화면 녹화를 지원합니다 (구현 예정 모듈 참고).
> 녹화 대상 URL과 클릭/입력 시퀀스는 큐레이션 단계에서 스크립트와 함께 생성됩니다.

### 제목 패턴
- `[시간/조건]에 [대상]으로 [결과]하는 법`
- 예: "자는 동안 제미나이 일 시켜서 100% 자동화하기"
- **숫자 포함 권장** (CTR 상승: %, 시간, 횟수)
- 길이: 25자 내외

### 주의
- **핵심 스텝은 반드시 시연** (따라 할 수 없을 정도로 생략 금지)
- **과장 훅 사용 시 본문에서 한계 짧게 언급** (장기 신뢰도)

## 포맷 선택 로직 (큐레이터 구현)

```
if 주제 == "신제품 / 신기능 / 툴 소개 / 개념 설명" → 포맷 A
elif 주제 == "실용 팁 / 활용법 / 워크플로우 / 튜토리얼" → 포맷 C
else → 포맷 A (디폴트)
```

### 제목 키워드 기반 분류 힌트
- **포맷 A 신호**: "출시", "공개", "등장", "업데이트", "뭐가 달라졌나", "이란"
- **포맷 C 신호**: "~하는 법", "~하기", "활용법", "~만에", "이렇게 쓰세요"

## 시리즈 연결 전략 (선택)

같은 주제에 대해 A → C 순서로 연작 제작 가능.

예시:
- **A편**: "Claude Skills, 이게 뭔가요"
- **C편**: "Claude Skills로 SKILL.md 만드는 법"

큐레이션 단계에서 주제 입력 시 "시리즈로 만들까요?" 옵션을 둘 수 있음.
구현은 추후 (우선순위 낮음).
