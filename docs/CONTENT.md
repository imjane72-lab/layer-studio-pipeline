## 🎨 브랜드 정체성
 
### Layer Studio (마스터 브랜드)
 
**핵심 메시지:** "한 층씩 파헤쳐 본질을 보여줍니다."
 
**차별화 포인트:** **한국인 오리지널 콘텐츠 + 글로벌 영어 자막**
 
**시각 정체성:**
- 컬러: 미니멀 블랙/화이트 + 채널별 액센트
- 폰트: 한국어 (Pretendard) + 영어 (Inter)
- 스타일: 깔끔, 전문적, K-감성
### 두 채널의 공통점
 
- 🎤 **음성:** 한국어 (Supertone Play로 복제한 내 음성 또는 직접 녹음)
- 📝 **자막:** 영어 (YouTube 공식 자막 등록)
- 🖼️ **제목/설명:** 영어 (YouTube 메타데이터)
- 📱 **포맷:** 9:16 faceless 쇼츠, 60초
- 🌐 **타겟:** 영어권 글로벌 + 한국 시청자

| **주제** | AI 빌더 관점 | K-Beauty 성분 과학 |

## 🤖 Layer AI Studio 가이드라인
 
### 핵심 포지셔닝
 
**"한국 개발자가 보는 글로벌 AI"**
 
- ❌ 투자자 관점 아님
- ❌ 그냥 뉴스 앵커 아님
- ✅ **skindit 개발한 빌더의 현장 관점**
- ✅ **한국 AI 씬 내부자의 시각**
### 콘텐츠 유형
 
**Type 1: Tool Launch Analysis (60%)**
- 새 AI 도구/모델 분석
- 빌더 관점 "이거 써볼만한가?"
- 예: "Claude Design, 빌더 입장에서 해부해봤다"
**Type 2: Technique Deep Dive (20%)**
- MCP, Agent SDK, RAG 등 기술 해부
- 한국어로 먼저 쉽게 설명, 영어 자막
- 예: "MCP가 에이전트 개발을 바꾸는 이유"
**Type 3: Builder Story (15%)**
- skindit 개발 경험 재활용
- "I built this in Korea" 스토리
- 예: "2주 만에 AI 스킨케어 앱 만든 썰"
**Type 4: Korean AI Scene (5%)**
- 한국 AI 시장 특수 관점
- 글로벌 시청자가 궁금해할 내용
- 예: "한국에서 Claude를 이렇게 쓴다"
### 스크립트 구조 (한국어 기준)
 
```
[0-3초] 훅: 질문 또는 놀라운 사실
  예: "문장 하나로 앱이 만들어진다면?"
 
[3-15초] 배경: 무슨 일이 일어났나
  예: "지난주 Anthropic이 Claude Design을 공개했다."
 
[15-45초] 핵심 인사이트 (빌더 관점)
  예: "개발자 입장에서 달라지는 건 이거다.
       원하는 걸 설명하면 Claude가 프로토타입을 만든다.
       목업이 아니다. 실제 작동하는 프로토타입이다."
 
[45-55초] 증거/예시
  예: "평소 3시간 걸리던 작업을 테스트해봤다.
       45초 만에 끝났다."
 
[55-60초] CTA (생각 유도)
  예: "디자이너에게 어떤 의미일까?"
```
 
### 톤 & 매너
 
**한국어:**
- 경어체는 기본이되 너무 딱딱하지 않게
- 짧은 문장 (한 문장 20자 내외)
- 1인칭 사용 가능 ("내가 써봤다")
- 전문 용어는 쓰되 과한 영어 섞기 지양
**영어 자막:**
- Direct and analytical
- 능동태
- Short sentences
- Preserve technical precision
### 좋은 예 vs 나쁜 예
 
**❌ 나쁜 예 (주식/투자 중심):**
> 한국어: "Anthropic 기업 가치가 200억 달러를 돌파했습니다.
>        1년 만에 10배 성장한 수치입니다.
>        투자자들이 몰리고 있습니다."
> 영어: "Anthropic's valuation hit $20 billion.
>       10x growth in one year.
>       Investors are piling in."
 
**✅ 좋은 예 (빌더 중심):**
> 한국어: "Anthropic이 Claude 5를 출시했다.
>        새 MCP 통합이 흥미롭다.
>        에이전트가 전체 코드베이스를 한 번에 읽는다.
>        사이드 프로젝트에 써봤다.
>        2시간 걸리던 게 3초다.
>        에이전트 개발의 새 기준선이다."
> 영어: "Anthropic shipped Claude 5.
>       The new MCP integration stands out.
>       Agents now read your entire codebase in one call.
>       I tried it on a side project.
>       Two hours became three seconds.
>       This is the new baseline for agent builders."
 
### 메타데이터 (영어)
 
**제목 (50-60자):**
- 좋은 예: "I Built an AI Agent in 10 Minutes with Claude SDK"
- 나쁜 예: "Amazing AI Tool You Must See"
**설명:**
- 첫 2줄 중요 (프리뷰)

## 📐 공통 편집 규칙
 
### 한국어 스크립트 길이
- **목표:** 380~420자 (약 60초)
- **최소:** 350자
- **최대:** 450자
### 한국어 문장 길이
- **평균:** 15-20자
- **최대:** 30자 (훅 제외)
- **최소:** 10자 (강조용)
### 영어 자막 문장 길이
- **평균:** 8-12 words
- **최대:** 15 words (모바일 가독성)
- **최소:** 5 words
### 매칭 원칙
- 한국어 1문장 = 영어 1문장 (1:1 매칭)
- 순서 동일하게 유지
- 복문 사용 자제 (분할 어려움)
- **문장 구분자(마침표/물음표/느낌표) 명확하게** — Supertone 문장별 분할 호출의 기초
### 인용/출처
- 특정 데이터 인용 시 출처 명시
- 한국어: "[출처]에 따르면"
- 영어 자막: "According to [출처]"
### 금지 표현 (한국어)
 
- ❌ "여러분 안녕하세요" (진부)
- ❌ "오늘 다룰 주제는..." (쇼츠에 안 맞음)
- ❌ "구독과 좋아요" (직접 CTA)
- ❌ "이런거 아시나요?" (청자 가정)
### 권장 표현 (한국어)
 
- ✅ "이거 흥미롭다."
- ✅ "사실은 이렇다."
- ✅ "중요한 건 이거다."
- ✅ "한번 생각해보자."
- ✅ "오늘 달라지는 건..."

## 🇰🇷 한국어 스크립트 작성 원칙
 
### 1. 영어 번역을 염두에 둔 작성
 
**좋은 구조 (번역 용이):**
```
Claude가 Claude Design을 공개했다.
빌더 관점에서 흥미로운 도구다.
원하는 걸 말하면 프로토타입이 나온다.
```
→ 각 문장 독립적, 1:1 영어 매칭 쉬움
 
**나쁜 구조 (번역 어려움):**
```
Claude가 Claude Design을 공개했는데, 이게 빌더 입장에서
흥미로운 도구인 게, 원하는 걸 말하면 프로토타입이 나오는
그런 식이기 때문이다.
```
→ 긴 복문, 영어로 자연스럽게 쪼개기 어려움
 
### 2. 대명사 최소화
 
**한국어에선 자주 생략되지만, 영어 번역에는 명시 필요:**
 
**원본:**
```
이거 써봤는데 좋더라.  ← 뭐를?
```
 
**개선:**
```
Claude Design을 써봤는데 좋더라.
```
 
### 3. 숫자/영어 단어 표기
 
**숫자:**
- 원칙: 아라비아 숫자 ("3시간", "45초")
- 10 이하 순서: 상황에 따라 ("첫 번째", "하나")
- **TTS 발음 주의**: "25살"처럼 네이티브 발음이 중요한 경우 "스물다섯 살"로 표기 고려
**영어 단어 섞기:**
- 자연스러운 건 OK ("API", "AI", "Claude")
- 과한 건 지양 ("이걸 deploy해야...")
- **TTS 발음 주의**: "GPT-5", "100%", 약어(MCP, SDK) 등은 Supertone 미리듣기로 발음 체크 후 확정
### 4. 문장 부호 (TTS 영향)
 
- **문장 끝: 마침표 필수** — Supertone 문장 분할의 기준점
- **쉼표**: 자연스러운 끊어 읽기 유도 (과하면 어색함)
- **느낌표/물음표**: 감정 표현 시 사용, Supertone이 인토네이션 반영
- **말줄임표(…) 주의**: TTS에서 긴 침묵이 되므로 짧은 쇼츠에선 비권장
### 5. Supertone 친화적 작성
 
**길게 쓰지 않기:**
- 한 문장 30자 넘으면 분할
- 문장이 짧을수록 Supertone 자연스러움 ↑
- 문장이 짧을수록 개별 재생성 비용도 낮음
**발음이 어색한 단어 대체:**
- 한자어보다 순우리말이 자연스러울 때 많음
- 브랜드명·약어는 한 번 미리듣기 필수

## 🇺🇸 영어 번역 원칙
 
### 1. 자연스러운 영어 우선
 
**❌ Literal 번역:**
```
한국어: "이거 흥미로운 도구다."
영어: "This is an interesting tool."  ← 밋밋함
```
 
**✅ Natural 번역:**
```
한국어: "이거 흥미로운 도구다."
영어: "Here's something interesting."  ← 자연스러움
```
 
### 2. 톤 유지
 
**AI 채널 (분석적):**
- 한국어: "이게 중요한 이유는..."
- 영어: "Here's why this matters..."
**Skin 채널 (따뜻한 과학):**
- 한국어: "피부가 고마워할 거예요."
- 영어: "Your skin will thank you."
### 3. 기술 용어
 
- 한국어 → 영어 상용어 사용
- "에이전트" → "Agent" (X "Secret Agent")
- "프로토타입" → "Prototype"
- "성분" → "Ingredient"
### 4. 한국 문화 맥락
 
**유지해야 할 한국적 표현:**
- "K-뷰티" → "K-Beauty" (글로벌 인지도)
- "식약처" → "Korean FDA (MFDS)"
- "한국에서는..." → "In Korea, we..."
**현지화해야 할 표현:**
- "대박" → "Game-changer"
- "진심" → "Honestly"
- "실화냐" → "Can you believe it?"
### 5. 모바일 가독성
 
- 한 자막에 15 단어 이내
- 쉽게 읽히는 단어 선택
- 긴 한 문장 → 영어 2문장으로 쪼개도 OK (시간만 맞추면)

## 🤖 프롬프트 템플릿
 
### 1. AI 채널 큐레이션 프롬프트
 
```
You are a content curator for Layer AI Studio.
 
Channel identity:
- Korean creator speaking Korean
- English subtitles for global viewers  
- Target: Korean AI devs + English-speaking AI enthusiasts
 
Select ONE news item best suited for a 60-second Short.
 
Criteria:
1. Builder Relevance - Practical for developers
2. Korean Insider Value - Does Korean perspective add something?
3. Global Appeal - Is this interesting outside Korea too?
4. Short-friendly - Can core fit in 60 seconds?
 
Reject:
- Pure investment/stock news
- Region-specific content that doesn't translate
- Duplicates: {recent_video_topics}
 
News items:
{news_items_json}
 
Output (JSON):
{
  "selected_id": "item_id",
  "reason": "Why this is best (Korean, 2-3 sentences)",
  "angle": "Core angle (Korean, 1 sentence)",
  "broll_keywords_en": ["keyword1", "keyword2", "keyword3"]
```
